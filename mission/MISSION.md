# 미션: "고객지원 챗봇 만들기"

## 상황

초록 코퍼레이션 백엔드 팀에 합류했습니다.

고객지원팀이 하루에 수백 건의 문의를 처리하고 있습니다.
같은 질문이 반복되고, 상담사는 지쳐가고, 대기 시간은 늘어납니다.

CTO가 슬랙 메시지를 보냈습니다:

> "데이터 서버에 자료 올려뒀어요. LLM API 키도 공유할게요.
> 고객 질문을 받아서 자동으로 답하는 REST API 만들어주세요.
> 이번 주 금요일까지요."

---

데이터는 여기 있습니다:

```
data/
├── layer1_faq/          # 공식 FAQ 문서 (영어)
├── layer2_policies/     # 사내 정책 문서 (영어, 버전 포함)
└── layer3_chatlogs/     # 고객 상담 로그 (한국어, JSONL)
```

그게 전부입니다. 추가 안내는 없습니다.

---

## 요구사항

### API 명세

```
POST /api/chat

요청:
{
  "question": "반품 기간이 얼마나 되나요?"
}

응답:
{
  "answer": "...",
  "tokenUsage": {
    "promptTokens": 0,
    "completionTokens": 0,
    "totalTokens": 0
  }
}
```

### 제약사항

- Spring AI의 `ChatClient`를 통해 OpenAI API를 사용해야 합니다
- 3개 레이어의 데이터를 **어떤 방식으로든** 활용해야 합니다
- 구현 방법에는 제약이 없습니다 — 알아서 해결하세요
- 토큰 사용량을 응답에 포함해야 합니다

### 완료 기준

- `POST /api/chat`이 200 응답과 함께 답변을 반환한다
- 테스트 질문 150개(`data/test_questions.json`)에 대한 정확도를 측정할 수 있다
- 3개 레이어 데이터를 모두 활용한다

그리고 한 가지 더 — **벽 리포트**:

```
mission/wall-report.md를 작성하세요:
- 구현하면서 부딪힌 한계와 문제
- 지금 방식으로 해결 못한 것
- 개선하고 싶은 것
```

## 시작하는 방법

`src/main/java/com/cholog/bootcamp/`를 열어보세요.

나머지는 직접 만들어야 합니다.

1. 미션 요구사항을 읽고 무엇이 필요한지 파악하세요
2. 필요한 클래스와 구조를 설계하세요
3. 하나씩 구현하고, 동작을 검증하세요
4. 만족할 때까지 개선하세요
