# 실라버스

이 과정은 Spring AI를 활용하여 "고객 지원 챗봇"을 직접 만드는 과정입니다.

예제를 따라가는 방식이 아니라, 실제 회사에서 마주칠 법한 데이터로 현장의 문제를 풀어간다는 점이 특징입니다.

---
## 시나리오

"고객지원 챗봇 만들어주세요." CTO의 슬랙 메시지가 옵니다.
데이터와 API 키가 있고, 추가 안내는 없습니다.

2주 뒤 당신 손에 남는 건 **동작하는 챗봇 하나**와, **그걸 만들면서 부딪힌 벽 목록**입니다.

---

## 2주 동안 일어나는 일

처음엔 뭘 어떻게 만들지 직접 결정합니다. 코드는 비어있습니다.

며칠 뒤 첫 답변이 나옵니다. 질문을 보내면 뭔가 답합니다.
근데 이상합니다. 구버전 정책이 섞이고, 상담 로그 노이즈가 올라옵니다.

테스트 질문 150개를 돌리면 정확도가 나옵니다.
Easy는 그럭저럭. Medium부터 급락합니다.

왜 틀리는지 분석합니다. 고칩니다. 다시 측정합니다.
오르기도 하고, 안 오르기도 합니다.

마지막에 벽 리포트를 씁니다.
"서버 재시작하면 데이터가 날아간다", "검색이 부정확하다", "비용을 모른다" —
내가 만든 것의 한계를 직접 정리합니다.

이게 전부 **의도된 경험**입니다.
벽의 이름을 아는 것, 왜 생기는지 아는 것 — 그게 이 2주의 목표입니다.

---

## 과정 목표
- Spring AI를 직접 써보며 작게라도 끝까지 만들어보는 경험하기
- 현장에서 발생하는 실제와 유사한 문제를 통해 왜 이런 문제가 발생했고 어떻게 해결하는지를 경험하기
- 만드는 과정에서 내가 뭘 알고 뭘 모르는지를 인지하기

## 기대 수행 시간

**하루 1시간 내외, 2주간** (총 10~14시간)

요구사항을 깊이 탐구하면 더 걸리고, 빠르게 구현에 집중하면 덜 걸립니다.
어느 쪽이든 괜찮습니다.
